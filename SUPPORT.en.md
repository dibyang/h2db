# Support

[中文](SUPPORT.md)

Use the project intake flow for h2db bugs, regressions, release questions, and feature requests:

* [Bug Reporting](docs/bug-reporting.en.md)
* [Contributing Guide](CONTRIBUTING.en.md)
* [Security Policy](SECURITY.en.md)

Before filing an issue:

* Search existing issues and upstream H2 documentation.
* Include the h2db version, Java version, operating system, and database URL options when relevant.
* Provide a small reproducible example for SQL, JDBC, MVStore, file locking, server, or web console problems.

For general H2 usage questions that are not specific to this fork, upstream H2 community resources such as the H2 mailing list and Stack Overflow may have more historical context.
